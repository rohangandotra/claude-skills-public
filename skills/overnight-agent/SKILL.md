# Overnight Agent

## Overview
Configures Claude Code for autonomous overnight operation with logging, checkpointing, and structured output so you wake up to organized results.

## When to Activate
- User says "run overnight", "work while I sleep", "autonomous agent"
- User has a large task that would take hours
- User wants to set up background agents

## Setup Process

### Step 1: Define the Task
Help the user specify:
- **Objective**: What should be done by morning? (e.g., "Research 50 competitors", "Generate 30 video scripts", "Audit all pages on my site")
- **Scope boundaries**: What should it NOT do? (e.g., "Don't modify any files", "Only research, don't execute")
- **Output location**: Where should results go? (default: `./overnight-output/`)
- **Checkpointing**: How often to save progress (default: every 10 items or 15 minutes)

### Step 2: Create the Agent Script

Generate a shell script that:

```bash
#!/bin/bash
# overnight-agent.sh
# Generated by overnight-agent skill

TASK_NAME="[task-description]"
OUTPUT_DIR="./overnight-output/$(date +%Y%m%d_%H%M%S)"
LOG_FILE="$OUTPUT_DIR/agent.log"

mkdir -p "$OUTPUT_DIR"

echo "Starting overnight agent: $TASK_NAME" | tee "$LOG_FILE"
echo "Output directory: $OUTPUT_DIR" | tee -a "$LOG_FILE"
echo "Started at: $(date)" | tee -a "$LOG_FILE"

# Run Claude Code in non-interactive mode with the task
claude --dangerouslySkipPermissions \
  --output-format json \
  -p "[detailed task prompt with checkpointing instructions]" \
  2>&1 | tee -a "$LOG_FILE"

echo "Completed at: $(date)" | tee -a "$LOG_FILE"
```

### Step 3: Configure Persistence

Provide instructions for keeping the agent running after terminal closes:

#### Option A: tmux (Recommended)
```bash
# Start a new tmux session
tmux new-session -d -s overnight-agent './overnight-agent.sh'

# Check on it later
tmux attach -t overnight-agent

# View output without attaching
tmux capture-pane -t overnight-agent -p
```

#### Option B: nohup
```bash
nohup ./overnight-agent.sh > overnight.log 2>&1 &
echo $! > overnight-agent.pid
```

#### Option C: launchd (macOS — survives restarts)
Generate a LaunchAgent plist:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.user.overnight-agent</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>[full-path]/overnight-agent.sh</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>StandardOutPath</key>
    <string>[full-path]/overnight-output/stdout.log</string>
    <key>StandardErrorPath</key>
    <string>[full-path]/overnight-output/stderr.log</string>
</dict>
</plist>
```

### Step 4: Output Structure

The agent should produce:

```
overnight-output/
  YYYYMMDD_HHMMSS/
    agent.log           # Full execution log
    summary.md          # High-level summary of what was done
    results/            # Task-specific output files
      item_001.md
      item_002.md
      ...
    errors.log          # Any failures or issues
    metrics.json        # Stats: items processed, time taken, tokens used
```

### Step 5: Morning Report

When the user returns, generate a summary:
```
## Overnight Agent Report
- Task: [description]
- Duration: [time]
- Items completed: [count]
- Errors: [count]
- Key findings: [top 3 bullet points]
- Files created: [list]
- Recommended next steps: [what to do with the output]
```

## Safety Rules
- Never make irreversible changes (no deleting, no pushing to git, no sending messages)
- Log every action taken
- If an error occurs 3 times on the same item, skip it and log the failure
- Stay within the defined scope boundaries
- Save progress frequently so nothing is lost if the process stops
